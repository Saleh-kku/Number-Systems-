8086 Microprocessor Tutorial

Introduction

The 8086 microprocessor, introduced by Intel in 1978, is one of the most influential processors in computer architecture history. It marked the transition from 8-bit to 16-bit processing and laid the foundation for modern microprocessors. This tutorial provides a detailed explanation of its features, architecture, and operations, serving as a comprehensive reference for students and professionals.

1. Microprocessor: Basics and Purpose

A microprocessor is the "brain" of a computer. It performs arithmetic, logic, and control operations, enabling the computer to execute instructions from a program. The 8086 microprocessor is a pivotal step in microprocessor evolution.

Why 8086?

  • The 8086 is an enhancement of the 8085 microprocessor, offering greater data width (16-bit) and addressing capabilities (1 MB memory).
  • It introduced pipelining, instruction queuing, and powerful instructions for operations like multiplication and division.

2. Features of the 8086 Microprocessor

The 8086 microprocessor includes several features that make it stand out. Let's explore them in detail:

  • 16-bit Processor: The 8086 has a 16-bit Arithmetic Logic Unit (ALU), 16-bit registers, and a 16-bit data bus. This means it can process 16 bits of data at once.
  • 20-bit Address Bus: With its 20 address lines, the 8086 can access 1 MB of memory (220 = 1,048,576 bytes).
  • Instruction Queue: The 8086 introduced pipelining, allowing it to prefetch up to 6 bytes of instructions. This reduces the CPU idle time and speeds up execution.
  • Dual Operating Modes:
    • Minimum Mode: For systems with a single processor.
    • Maximum Mode: For multiprocessor systems.
  • Clock Speed: Available in three versions based on clock speed:
    • 8086: 5 MHz
    • 8086-2: 8 MHz
    • 8086-1: 10 MHz
  • Interrupts: Supports 256 vectored interrupts, allowing it to handle multiple I/O devices efficiently.
  • Transistors: Contains 29,000 transistors, showcasing its complexity compared to earlier microprocessors.

4. Architecture of the 8086 Microprocessor

The architecture of the 8086 is divided into two main units:

  • Bus Interface Unit (BIU): Handles all data transfer between the processor and memory/I/O.
  • Execution Unit (EU): Executes instructions fetched by the BIU.

For a detailed breakdown of these units, refer to Functional Block Diagram.

5. Instruction Set of the 8086

The 8086 has a rich set of instructions classified into categories:

  • Data Transfer Instructions: Moves data between registers, memory, and I/O.
  • Arithmetic Instructions: Performs addition, subtraction, multiplication, etc.
  • Logical Instructions: Performs AND, OR, XOR, NOT operations.
  • Control Transfer Instructions: Alters the flow of execution.
  • String Instructions: Operates on blocks of data.

6. Pipelining in the 8086

Pipelining is a technique used to improve the performance of the microprocessor by overlapping instruction execution. It consists of two main stages:

  • Fetch Stage: The BIU fetches instructions and stores them in the instruction queue.
  • Execute Stage: The EU decodes and executes instructions in the queue.

7. Interrupts in the 8086

Interrupts temporarily halt the normal execution of a program to handle an event. The 8086 supports 256 vectored interrupts for efficient multitasking.
